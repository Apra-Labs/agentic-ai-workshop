The LLM Has No Memory — That Changes Everything 4 min

For the LLM, every request is a fresh start. There is no conversation — only the illusion of one.

The Fundamental Truth

Text in, text out. Nothing persists between calls.

The Conversation Illusion

[System prompt]
>> You: msg1
<< Agent: rsp1
>> You: msg2
<< Agent: rsp2
>> You: msg3
<< Agent: rsp3
...
ALL of this is re-sent. EVERY turn.

Context Window = Text Budget

~200k tokens for Claude

When full → compaction (older messages get summarized)

What Forgetting Looks Like:

  • Agent re-asks a question you answered
  • Agent contradicts earlier work
  • Agent “forgets” a convention

.md files = memory that survives sessions

THIS is the foundational slide. If they don’t get this, nothing else makes sense. Ask who thought the agent “remembers.” The packing luggage metaphor helps. “/clear is freeing, not scary — it gives you a fresh context.”

ONBOARD — Give the Agent Context Before You Give It Work 10 min

Why Onboarding Matters

The agent knows nothing about your project. Before you give it work, give it context.

This is the realistic scenario: you joined a team, there’s existing code, now build on it.

What happens during onboard:

  • Agent reads API routes, models, tests
  • Agent discovers endpoints, data shapes, validation rules — WITHOUT being told
  • CLAUDE.md gets updated with frontend context

Onboarding is the most underrated step. Skip it and every instruction you give lacks context.

Start Supervised — Earn Your Autonomy 5 min

The Four Levels

ModeWhat It MeansRiskReward
NannyAgent asks permission for every operationNear zeroNear zero
AllowlistedOnly pre-approved commands runLowModerate
SandboxedOS-level filesystem + network boundariesLowHigh
YOLO + DenylistEverything runs except blocked commandsHighMaximum

Most reject AI after Nanny mode. Some skip to YOLO. Both are mistakes.

Today’s Lesson

Today I ran in supervised mode. You saw me approve every action.

When to graduate:

  • You trust the branch discipline
  • You’ve seen the agent’s patterns
  • You have tests that catch regressions
  • You’re committing before risky operations

You can only denylist what you’ve learned to fear.

The agent has terminal access. It can install packages, run scripts, make network calls. Supervision is not paranoia — it’s engineering.

“In the live build, I ran in supervised mode. Every file write, every npm install, I approved. That’s intentional for a first project.” Connect to advanced workshop: “The advanced workshop shows you how to move to overnight autonomy safely — hooks, sandboxing, the full stack.”

Guardrails Checklist — Your Safety Net 3 min

The Checklist

  • Branch discipline: never code on main/development
  • Test integrity: agent can’t delete/skip tests
  • Commit before risk: git checkpoint before big changes
  • Commit discipline: small, meaningful commits
  • Stuck protocol: if stuck 3 attempts, ask human
  • Review before merge: read the diff, not just the tests
  • /clear between phases: fresh context for each phase

Print this. Use it Monday.

These aren’t suggestions. They’re the minimum for responsible agentic development.

What happens without guardrails:

  • Agent deletes a failing test to make suite pass
  • Agent installs unvetted packages
  • Agent overwrites manual changes
  • Agent commits directly to main/development

Every one of these has happened in real projects.

“This is the most practical slide. Print the checklist handout. Tape it next to your monitor. Every item on this list comes from a real failure — either mine or documented in the community.” Commit discipline added per Kashyap’s feedback.
